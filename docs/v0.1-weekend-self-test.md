# MyRhythm Founding Edition (v0.1) — Weekend Self-Test Script

**Audience:** you (founder), Saturday morning.
**Time:** ~30 minutes end-to-end.
**Goal:** prove every Founding Core surface works for a real user under cognitive load, and that every output can be copied, amended, and emailed.

Sit somewhere quiet. Have a phone or second tab open for the "send email" steps.

---

## 0 · Pre-flight (5 min)

- [ ] Open the preview URL in a private/incognito window.
- [ ] Sign out if signed in. Clear `localStorage` for the domain.
- [ ] Confirm seed test account credentials are to hand.
- [ ] Have a real email address ready (one you can check on phone).

**If anything breaks here →** check `src/integrations/supabase/client.ts` and `.env` for `VITE_SUPABASE_*` values.

---

## 1 · Public funnel (5 min)

- [ ] Visit `/`. Confirm single H1, no medical claims, "Founding Edition" badge visible.
- [ ] Click into `/mvp/user-type-selection`. Pick **Brain Injury Recovery**.
- [ ] Walk through `/mvp/assessment`. Confirm the no-diagnosis disclaimer is present on every step.
- [ ] Land on `/mvp/payment`. Use test payment path (do not complete real charge).
- [ ] Arrive at `/launch/welcome`. Confirm EditionBadge sits above the headline.

**If this breaks →** check `src/App.tsx` routing block (lines ~376–408) and `src/components/landing/`.

---

## 2 · The 4C loop with real output handling (12 min)

This is the test that matters most for the weekend.

### Capture
- [ ] Go to `/launch/memory`. Tap the mic. Record a 45–60s message like:
  > "Reminder for myself — call the GP on Monday about repeat prescription. Also pick up Sarah from the station at 6pm Friday. And book the dentist."
- [ ] Stop. Name the recording "Self-test Saturday".
- [ ] Tap **Save & Extract Actions**.
- [ ] When the dialog appears, choose **Review individually**.

**Output check (🟥 critical):**
- [ ] In the recording row, the new **Copy / Email / Download** toolbar is visible.
- [ ] Tap **Copy** → paste into a note, confirm you see title + date + summary text.
- [ ] Tap **Email** → confirm your mail client opens with subject + body pre-filled.
- [ ] Tap **Download** → confirm a `.txt` file lands in Downloads.

### Commit
- [ ] Go to `/launch/commit`. Confirm at least one of the extracted actions appears.
- [ ] Use the **Copy** action on the day's plan → paste into a note, confirm a clean list.
- [ ] Mark one action complete.

### Calibrate
- [ ] Go to `/launch/calibrate`. Add a one-line reflection ("Made my calls without forgetting").
- [ ] Use **Email** to send the reflection to yourself.

### Celebrate
- [ ] Confirm a small win appears on `/launch/home` in the Today's Wins panel.

**If extraction fails →** check Supabase Edge Function logs for `extract-acts-incremental`.

---

## 3 · Memory Library access (3 min)

- [ ] Open `/launch/memory` again.
- [ ] Find "Self-test Saturday" in the recordings list.
- [ ] Tap **Play** → audio plays back.
- [ ] Tap **View Actions** → action sheet opens with all extracted items.
- [ ] Confirm Copy/Email/Download are present on the recording row.

**Note:** the centralised, filterable Memory Library (search by quote, filter by date, starred view) is the next deliverable in v0.1.1. For weekend self-test, the recordings list on `/launch/memory` is the single source of truth.

---

## 4 · Support Circle (3 min)

- [ ] Go to `/launch/support`. Invite a fake email (`test+sc@yourdomain`).
- [ ] Confirm the invitation appears in pending state.
- [ ] Toggle `can_record_on_behalf` ON for a member, then OFF. Confirm it persists on refresh.
- [ ] Confirm the **"Record meeting for [name]"** button appears only when the toggle is ON.

**If invite fails →** check `send-support-invitation` edge function logs.

---

## 5 · Safety net (2 min)

- [ ] On `/launch/memory` and `/launch/calibrate`, confirm the no-medical-claims disclaimer text is present.
- [ ] On `/launch/settings`, trigger **Download my data** → confirm both JSON and PDF are produced.
- [ ] If you have a real meeting recorded, generate a **Clinical Export** PDF → confirm the footer reads `Founding Edition · v0.1` plus the confidentiality line.
