## B. Founding Core (shipping to the cohort)

Nine routes. Each earns the £7–£20/mo price by removing a concrete, expensive pain.

| Route | One-line purpose | Key components | Why it earns the price |
|---|---|---|---|
| `/launch/home` | 4C loop in one screen | `LaunchDashboard`, `MVPTopNav` | Every day starts and ends in one calm place — no app-hunting under cognitive load. |
| `/launch/capture` | One-tap voice/text capture | `LaunchCapture`, `VoiceRecorder` | Replaces £40/hr private-OT note-taking and the 3am "what did the consultant actually say?" panic. |
| `/launch/commit` | Turn captures into committed actions | `LaunchCommit`, `EnhancedActionCard` | Stops the 70%+ drop-off between "good intention" and "thing actually done." |
| `/launch/calibrate` | Schedule against real energy | `LaunchCalibrate`, energy badges | Prevents the crash-and-burn cycle that costs working days and family time. |
| `/launch/memory` | Memory Bridge — searchable record | `LaunchMemoryBridge`, `LaunchCaptureResult` | One trustworthy place; no losing the clinical conversation that mattered. |
| `/launch/calendar` | Day-of cognitive-load view | `LaunchCalendar` | Plans the day a brain-injured / ADHD / burnt-out user can actually execute. |
| `/launch/support` | Support Circle invites + permissions | `LaunchSupportCircle`, `InviteFlow` | Family/clinician informed without the user repeating themselves; nothing auto-shared. |
| `/launch/settings` | Retention, MFA, calendars, edition, feedback | `LaunchSettings`, `LaunchEditionAbout` | Trust controls in plain sight — privacy, MFA, 30-day voice retention, direct feedback to founder. |
| `/launch/profile` | Identity + persona | `LaunchProfile` | Persona-correct experience across all 9 founding personas. |

Public funnel exposed alongside the Core: `/`, `/mvp/user-type-selection`, `/mvp/assessment`, `/mvp/payment`, `/auth`, `/privacy-policy`, `/terms`.

## C. Hidden behind `FOUNDING_CORE_ONLY` flag (queued for v0.2)

Built, tested-light, sequenced out of the cohort to keep the value story tight.

| Route | Reason for deferral |
|---|---|
| `/launch/games` | Nice-to-have; not load-bearing on the value promise. |
| `/launch/store` | Pricing/packaging not finalised for add-ons. |
| `/launch/analytics` | Needs live signals before it earns trust. |
| `/launch/roadmap` | Move to a static marketing page for v0.1. |
| `/launch/science` | Move to marketing site; risk of medical-claim drift. |
| `/launch/whats-new` | Empty at launch; reopen after first shipped update. |
| `/launch/clinical-brief` | Needs real PDF + clinician review before exposure. |
| `/launch/continuity` | Operating layer is implicit in the 4C loop for v0.1. |
| `/launch/vision-statement` | Folds into Goals workspace in v0.2. |
| `/launch/goals` | Re-enable once Traceability model (Vision → Goals → Priorities → Actions) is wired end-to-end. |
| `/launch/gratitude` | Surface via Calibrate prompt instead of standalone route. |
| `/launch/sc/capture/*` | Re-enable with Support Circle v2 (permissions matrix). |
| `/launch/settings/edition` | Kept reachable only as a link inside Settings, not as a top-level nav. |
