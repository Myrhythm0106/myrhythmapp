## 3. Daily loop (Day 1+) — the 4C walk

The cohort never has to think about which screen to open next. Home points to the one next action.

```text
        ┌─────────────────────────────────────────────────┐
        │  /launch/home                                    │
        │  • today's energy badge                          │
        │  • 1 next action (Capture | Commit | Calibrate)  │
        │  • link to Memory Bridge                         │
        └───────────────┬─────────────────────────────────┘
                        │
        ┌───────────────▼─────────────┐
        │  /launch/capture            │  voice or text, one tap
        │  → upload + transcribe      │  writes voice_recordings,
        │                             │         transcription_jobs
        └───────────────┬─────────────┘
                        │ transcript ready
        ┌───────────────▼─────────────┐
        │  /launch/memory/result/:id  │  summary + extracted actions
        │  OutputActions: Copy/Email  │  writes extracted_actions
        │       /Download/Edit        │
        └───────────────┬─────────────┘
                        │ "Commit this"
        ┌───────────────▼─────────────┐
        │  /launch/commit             │  pick actions, owner, due date,
        │  energy fit badge           │  energy fit
        └───────────────┬─────────────┘  writes calendar_events
                        │
        ┌───────────────▼─────────────┐
        │  /launch/calendar           │  day-of cognitive-load view
        └───────────────┬─────────────┘
                        │ end of day
        ┌───────────────▼─────────────┐
        │  /launch/calibrate          │  energy reflection + gratitude
        │  prompt (folded in)         │  writes mood_entries,
        │                             │         gratitude_entries
        └───────────────┬─────────────┘
                        │
                  next day's energy estimate
                        │
                        ▼
                  back to /launch/home
```

`/launch/memory` is the always-on index. The user can drop in from any 4C step to find a past recording, summary, or action.

## 4. Re-entry safety (cognitive load assumed)

| Situation | What v0.1 does |
|---|---|
| App reopened mid-day | `/launch/home` re-renders today's energy + one next action. No modal. |
| Recording interrupted (call, lock screen) | Capture page resumes with "Continue last recording" or "Start fresh". No silent loss. |
| Transcription job still pending | `/launch/memory/result/:id` shows a friendly "Still processing — we'll keep this safe" state with retry link. |
| User on a low-energy day | Home shows fewer items; Calibrate auto-suggests a lighter schedule. |
| User opens a legacy URL (`/dashboard`) | Planned redirect → `/launch/home` (not yet wired — see §7). |
| MFA challenge required | Settings flow; on success returns user to last intended Core route. |

## 5. People & trust flow

```text
/launch/support
   ├─ Invite by email (Family | Clinician | Friend)
   ├─ Permission per invite (View summaries | View actions | Comment)
   ├─ Nothing auto-shared — user explicitly forwards from Memory Bridge
   └─ Writes: support_circle_members

/launch/settings
   ├─ Voice retention (default 30 days, reminders at day 5 + day 25)
   ├─ MFA enrol / unenrol
   ├─ Calendar links (Google, Outlook) — read-only sync in v0.1
   ├─ Edition transparency link → /launch/settings/edition
   └─ Direct feedback to founder → founding_feedback

/launch/profile
   ├─ Display name, persona, timezone
   └─ Writes: profiles
```

**Never shared without explicit user action:** raw recordings, transcripts, mood entries, gratitude entries, assessment answers.

## 9. Failure & offline behaviour (v0.1 minimum)

| Failure | User-visible behaviour |
|---|---|
| Mic permission denied | Capture screen shows plain-language fix steps + text fallback. |
| Network drops mid-upload | Recording held in browser; retry CTA on Capture. No silent loss. (Full offline queue is v0.2.) |
| Transcription still pending | Memory result shows "still processing" with last-updated time. Manual retry link. |
| Edge function error | Toast in plain language: "We couldn't process that — your recording is safe. Try again." |
| Stripe/Paddle failure at payment | Stays on `/mvp/payment` with retry; no half-created account. |
